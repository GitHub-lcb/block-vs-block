# 方块战方块 - 双人对战微信小游戏

## 项目概述

开发一款双人实时对战的微信小游戏，融合俄罗斯方块和弹球两种经典游戏模式，通过30秒角色互换机制创造独特的博弈体验。

### 核心玩法特色

- **共享场地对战**：双方在同一个游戏场地内实时对抗
- **深度互动**：方块落地变砖块 → 弹球击碎；消除满行 → 场地下移
- **角色互换**：每30秒强制切换，场地重置，策略规划与临场反应并存
- **节奏切换**：每个回合独立，分数累计，互换成为节奏切换点

## 第一部分：游戏玩法设计

### 1.1 共享场地机制

#### 场地结构

场地从上到下分为四个层次：

| 层级 | 名称 | 内容 |
|------|------|------|
| 空中区域 | 方块下落区 | 方块从顶部下落，球在此区域反弹 |
| 砖块区域 | 弹球目标区 | 可被弹球击碎的砖块 |
| 挡板区域 | 弹球控制区 | 玩家控制挡板接球 |
| 堆叠区域 | 方块堆积区 | 方块落地后变成堆叠，球可击碎 |

#### 双方互动规则

**方块方操作的影响**：
- 方块落地 → 变成场地内的砖块，弹球可以击碎
- 消除满行 → 场地整体下移，弹球活动空间增大（利好双方）
- 堆叠过高 → 弹球反弹空间减少（风险共担）

**弹球方操作的影响**：
- 击碎砖块 → 清除方块堆叠，为方块方腾出空间
- 球落出底部 → 失去一球，扣减总分

#### 胜负判定

| 条件 | 结果 |
|------|------|
| 方块堆叠到顶部 | 方块方失败，弹球方获胜 |
| 球丢失次数过多 | 弹球方分数大幅扣减 |
| **3分钟时间结束** | 总分高者获胜 |

**游戏时长**：每局 **3分钟**，共 **6个回合**（每回合30秒）。

<!-- CEO评审补充：碰撞规则 -->
#### 碰撞规则（共享场地）

| 场景 | 规则 | 说明 |
|------|------|------|
| 球与下落方块 | **穿透** | 下落中的方块对球透明，球可以直接穿过 |
| 球与已落地砖块 | **碰撞** | 落地后的方块变成砖块，球可以击碎 |
| 球与场地边界 | **反弹** | 球碰到场地边缘反弹 |
| 方块与球 | **无影响** | 方块下落不受球影响 |

**设计意图**：简化实时碰撞检测复杂度，保持双方操作的独立性。弹球玩家专注于击碎已落地的砖块，而非"干扰"正在下落的方块。

### 1.2 俄罗斯方块玩家（攻击方）

#### 基础操作

| 操作 | 触控方式 | 说明 |
|------|----------|------|
| 左移 | 点击左区 | 方块向左移动一格 |
| 右移 | 点击右区 | 方块向右移动一格 |
| 旋转 | 点击中区 | 方块顺时针旋转 |
| 软降 | 下滑 | 加速下落 |
| 硬降 | 快速下滑 | 瞬间落底 |
| Hold | 双击 | 暂存当前方块 |

#### 计分规则

| 消除行数 | 基础分 | T-Spin加成 |
|----------|--------|------------|
| 1行 | 100 × 等级 | T-Spin: 800 × 等级 |
| 2行 | 300 × 等级 | T-Spin Double: 1200 × 等级 |
| 3行 | 500 × 等级 | T-Spin Triple: 1600 × 等级 |
| 4行 | 800 × 等级 | - |

**T-Spin检测规则**：按照标准 Tetris Guideline 规则，使用 **LSRP (Last Success Rotation Position)** 检测。当 T 方块落地前最后一次操作是旋转，且落地后四个角落至少有3个被占据，则判定为 T-Spin。

**额外奖励**：
- Back-to-Back（连续高难度消除）：分数 × 1.5
- Perfect Clear（场地清空）：额外 +800~2000 分

<!-- CEO评审补充：等级系统 -->
#### 等级系统

| 规则 | 说明 |
|------|------|
| 升级机制 | **时间驱动**：每60秒自动升级1级 |
| 等级影响 | 方块下落速度提升（**不影响弹球玩家**） |
| 最高等级 | 15级 |

**设计意图**：采用时间驱动升级而非传统俄罗斯方块的"消除行数"升级，确保双方在相同时间内获得相同的等级优势，保持公平性。等级只影响方块下落速度，弹球玩家不受等级影响，避免双方难度不平衡。

#### 特殊机制

- **Ghost Piece**：显示方块落点预览
- **Next 预览**：显示接下来3个方块
- **Hold 系统**：暂存当前方块，最多使用1次

### 1.3 弹球玩家（破坏方）

#### 基础操作

| 操作 | 触控方式 | 说明 |
|------|----------|------|
| 挡板左移 | 点击左区 | 挡板向左移动 |
| 挡板右移 | 点击右区 | 挡板向右移动 |
| 发射球 | 点击中区 | 发射球 |
| 加速球 | 长按 | 球速提升50% |

#### 计分规则

| 行为 | 分数 | 说明 |
|------|------|------|
| 击碎普通砖块 | +1 分 | 基础得分 |
| 击碎铜砖 | +2 分 | 需击中2次 |
| 击碎金砖 | +5 分 | 稀有砖块 |
| 连击加成 | +连击数 | 连续击中不落球 |

#### 球生成与丢失机制

- 球落出屏幕后，5秒自动生成新球
- 每局最多丢失3球
- **超过3球后扣分**：每次丢失球扣减 10 分
- 3球全部丢失后游戏继续，但继续丢球会持续扣分

#### 球速度

| 规则 | 说明 |
|------|------|
| 初始速度 | 基础速度（固定） |
| 速度增长 | **不随等级变化** |
| 加速球 | 长按可提升50%速度（临时） |

**设计意图**：球速度固定，弹球玩家难度不随游戏进程变化，让玩家专注于反应和操作。方块方难度随等级提升（下落加速），双方难度曲线不同，增加策略性。

### 1.4 角色互换系统

#### 互换规则

```
┌─────────────────────────────────────────────────────────┐
│  30秒游戏回合                                            │
├──────────────┬──────────────┬──────────────┬────────────┤
│  0-25秒      │  25-30秒     │  互换动画    │  新回合    │
│  正常游戏    │  闪烁提示    │  0.5秒       │  角色切换  │
└──────────────┴──────────────┴──────────────┴────────────┘
```

#### 互换时状态处理

**场地重置**：
- 场地内所有砖块和堆叠 → **清空**
- 当前下落的方块 → **消失，重新生成新方块**
- 球的位置 → **重置到挡板上方**

**状态归零**：
- 连击数 → **归零**（新回合重新积累）
- 道具效果 → **清除**（护盾、加速等效果消失）

**保留状态**：
- 双方得分 → 保留
- 剩余球数 → 保留
- 道具库存 → 保留（已获得但未使用的道具）

#### 互换后新回合

| 操作 | 说明 |
|------|------|
| 场地初始化 | 空场地，新方块从顶部生成 |
| 球重置 | 新球在挡板上方等待发射 |
| 角色切换 | P1/2 角色互换，控制方式自动切换 |

**设计意图**：每次互换后场地重置、连击归零、道具效果清除，确保双方在相同起跑线上开始新回合。分数累计保留，激励玩家在每个回合争取更高得分。互换成为节奏切换点而非博弈点，降低策略复杂度，让玩家专注于当前回合的操作。

### 1.5 道具系统

| 道具 | 效果 | 触发条件 |
|------|------|----------|
| ⚡ 加速 | 对手下落速度+50%（持续10秒） | 消除4行 |
| 🛡️ 护盾 | 免疫一次球丢失 | 连击≥5 |
| 🎯 瞄准 | 显示球落点预测线（持续15秒） | 击碎金砖 |
| 🧱 加固 | 底部生成保护层（持续到互换） | Perfect Clear |

**道具状态说明**：
- **道具库存**：已获得但未使用的道具，互换后保留
- **道具效果**：已激活的道具效果（加速中、护盾激活等），互换后清除

**设计意图**：互换后清除道具效果，确保每个回合都是公平的新开始。道具库存保留，让玩家可以策略性地选择使用时机。

### 1.6 特殊方块与砖块

#### 俄罗斯方块特殊方块

| 方块 | 出现概率 | 效果 |
|------|----------|------|
| 💎 钻石方块 | 5% | 消除时额外+500分 |
| 💣 炸弹方块 | 3% | 落地后消除周围3×3格子 |
| 🌈 彩虹方块 | 2% | 可变成任意形状 |

#### 弹球特殊砖块

| 砖块 | 颜色 | 耐久 | 效果 |
|------|------|------|------|
| 普通 | 白/绿/蓝 | 1 | 基础砖块 |
| 铜砖 | 🟤 棕色 | 2 | 需击中2次 |
| 金砖 | 🟡 金色 | 1 | +5分，掉落道具 |
| 钢砖 | ⬜ 灰色 | ∞ | 不可破坏，**初始布局生成** |
| 炸弹砖 | 🔴 红色 | 1 | 爆炸消除周围砖块 |

**钢砖说明**：钢砖作为场地障碍物，在每回合开始时随机生成若干个（**数量 = 等级 × 2**），不会被弹球击碎，增加游戏难度。互换后场地重置，钢砖重新随机生成。

### 2.2.1 用户情感旅程

#### 一局游戏情感弧线（3分钟）

| 阶段 | 时间 | 用户行为 | 情绪目标 | 设计支持 |
|------|------|----------|----------|----------|
| **期待** | 主菜单 | 点击「开始游戏」 | 兴奋、好奇 | 醒目的开始按钮 |
| **沉浸** | 加载中 | 等待游戏启动 | 耐心、期待 | 方块下落加载动画 |
| **专注** | 0-25秒 | 操作方块/弹球 | 专注、投入 | 清晰的当前角色标识 |
| **紧张** | 25-30秒 | 看到互换警告 | 紧张、紧迫 | 闪烁警告 + 倒计时 |
| **释放** | 互换动画 | 观看动画 | 放松、好奇 | 流畅的互换动画 |
| **适应** | 新回合 | 操作新角色 | 切换、专注 | 明确的角色切换提示 |
| **...** | 循环3次 | ... | ... | ... |
| **高潮** | 最后30秒 | 争夺胜利 | 激烈、刺激 | 更快的倒计时节奏 |
| **结果** | 结算 | 查看结果 | 满足/遗憾 | 详细的数据展示 |
| **留存** | 结算后 | 选择「再来一局」 | 再来一局的冲动 | 突出的再来一局按钮 |

#### 关键情绪设计

**互换紧张感**（核心体验）：
- 25秒时警告闪烁 → 产生紧迫感
- 倒计时显示 → 量化剩余时间
- 当前操作突然变得「重要」 → 情绪聚焦

**胜负高潮**：
- 最后30秒倒计时加速 → 情绪升温
- 分数差距缩小 → 悬念
- 结算时详细数据 → 成就感/复盘价值

#### 响应式与无障碍规范

### 5.5 性能优化策略

#### 对象池技术

| 对象类型 | 池化策略 |
|----------|----------|
| 砖块 | 预创建50个，复用消除后的砖块 |
| 方块 | 预创建10个（7种 + 备用） |

#### 渲染优化

| 优化项 | 策略 |
|--------|------|
| 批量渲染 | 同类型砖块合并为单个DrawCall |

| 裁剪优化 | 场地外对象不渲染 |
| 纹理合图 | 所有游戏元素合为单张图集 |

| 优化项 | 策略 |
|--------|------|
| 碰撞检测 | 使用空间哈希网格，O(1)查询 |
| 物理更新 | 固定时间步长，避免累积误差 |
| 状态机 | 状态变化时才更新，而非每帧检查 |

#### 内存管理

| 优化项 | 策略 |
|--------|------|
| 资源卸载 | 互换动画时卸载上一回合资源 |
| 内存监控 | 每分钟检查内存，超过150MB触发GC |
| 缓存策略 | 常用资源常驻内存，特效按需加载 |
